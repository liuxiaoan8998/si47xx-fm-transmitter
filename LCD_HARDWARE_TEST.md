# Si47xx 硬件测试程序 (3位LCD屏版)

基于 3 位断码屏的 Si47xx FM 发射器硬件测试程序，通过显示错误码快速定位问题。

## 错误码定义

### 1xx - I2C 通信错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 101 | I2C_WRITE_ERROR | I2C 写操作失败 | 检查 SDA/SCL 连接，确认地址 0x21 |
| 102 | I2C_READ_ERROR | I2C 读操作失败 | 检查 SDA/SCL 连接，确认芯片供电 |
| 103 | I2C_NO_RESPONSE | 芯片无响应 | 检查 RST、SEN 引脚，确认芯片供电 |
| 104 | I2C_TIMEOUT | I2C 通信超时 | 检查 I2C 速度，可能太快 |

### 2xx - 芯片初始化错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 201 | CHIP_INIT_FAILED | 芯片初始化失败 | 检查复位序列，确认芯片型号 |
| 202 | CHIP_POWERUP_FAILED | POWER_UP 命令失败 | 检查 RST 引脚时序，确认电源稳定 |
| 203 | CHIP_GET_REV_FAILED | 无法获取芯片版本 | 检查 I2C 通信，芯片可能损坏 |
| 204 | CHIP_NOT_DETECTED | 芯片未检测到 | 检查芯片型号，确认连线正确 |

### 3xx - 频率设置错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 301 | FREQ_OUT_OF_RANGE | 频率超出范围 | 检查频率是否在 87.5-108MHz 范围 |
| 302 | FREQ_SET_FAILED | 频率设置失败 | 检查芯片是否支持该频率 |
| 303 | FREQ_INVALID | 无效频率 | 频率格式错误或超出范围 |
| 304 | FREQ_TUNING_FAILED | 调谐失败 | 检查天线连接，可能超出覆盖范围 |

### 4xx - 功率设置错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 401 | POWER_OUT_OF_RANGE | 功率超出范围 | 功率应在 77-115 dBuV 范围 |
| 402 | POWER_SET_FAILED | 功率设置失败 | 检查芯片是否支持该功率 |
| 403 | POWER_INVALID | 无效功率值 | 功率值格式错误 |

### 5xx - RDS 功能错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 501 | RDS_INIT_FAILED | RDS 初始化失败 | 检查芯片型号，可能不支持 RDS |
| 502 | RDS_SET_FAILED | RDS 设置失败 | RDS 功能异常 |
| 503 | RDS_UNSUPPORTED | RDS 不支持 | 芯片型号不支持 RDS 功能 |

### 6xx - 音频控制错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 601 | AUDIO_VOL_SET_FAILED | 音量设置失败 | 检查音频输入连接 |
| 602 | AUDIO_MUTE_FAILED | 静音设置失败 | 音频控制异常 |

### 7xx - 属性设置错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 701 | PROP_SET_FAILED | 属性设置失败 | 检查属性 ID 和值的有效性 |
| 702 | PROP_GET_FAILED | 属性读取失败 | 检查属性 ID 是否可读 |

### 8xx - 配置错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 801 | CONFIG_INVALID | 无效配置 | 检查配置参数 |
| 802 | CONFIG_MISSING | 配置缺失 | 检查必要参数是否设置 |

### 9xx - 系统错误
| 错误码 | 含义 | 可能原因 | 解决方法 |
|--------|------|----------|----------|
| 901 | SYS_MEM_ERROR | 内存错误 | 系统内存不足 |
| 902 | SYS_TIMEOUT | 系统超时 | 检查系统负载 |

## 使用方法

### 1. 硬件准备
```c
// 需要实现以下函数
void LCD_dispnum(char number);  // 显示单个数字 '0'-'9'
void si47xx_delay_ms(uint32_t ms);  // 毫秒延时
int si47xx_hw_write(uint8_t addr, const uint8_t *data, uint16_t len);  // I2C写
int si47xx_hw_read(uint8_t addr, uint8_t *data, uint16_t len);  // I2C读
```

### 2. 完整测试
```c
#include "lcd_test_si47xx.c"

int main() {
    int error_code = si47xx_hardware_test();
    // 错误码会显示在3位LCD屏上
    return 0;
}
```

### 3. 快速测试
```c
int main() {
    int error_code = si47xx_quick_test();
    // 错误码会显示在3位LCD屏上
    return 0;
}
```

## 显示格式

### 测试阶段显示
- `001` - I2C通信测试
- `002` - 芯片初始化测试
- `003` - 频率设置测试
- `004` - 功率设置测试
- `005` - RDS功能测试
- `006` - 音频控制测试
- `007` - 属性设置测试

### 错误码显示
- `101`, `102`, `103`... - I2C通信错误
- `201`, `202`, `203`... - 芯片初始化错误
- 依此类推...

### 测试通过
- `000` - 所有测试通过

## 故障诊断流程

1. **观察错误码** - 记录LCD显示的3位数字
2. **查找含义** - 对照上面的错误码表
3. **定位问题** - 根据可能原因检查硬件
4. **解决问题** - 按解决方法操作
5. **重新测试** - 再次运行测试程序

## 特殊情况

- **RDS 不支持** - 显示 `050` 表示芯片不支持RDS，非错误
- **测试通过** - 显示 `000` 表示所有功能正常
- **无显示** - 检查LCD接口连接

## 集成到项目

1. 复制 `lcd_test_si47xx.c` 到你的项目
2. 实现必要的硬件接口函数
3. 调用 `si47xx_hardware_test()` 或 `si47xx_quick_test()`
4. 观察LCD显示屏上的错误码
